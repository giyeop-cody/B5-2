# 기능 요구 사항 (Functional Requirements)

다음 요구사항을 모두 만족해야 한다.

## 1. 서버 실행
- Python 가상환경을 생성하고, 필요한 패키지를 설치한다.
  - **필수 패키지:** `fastapi`, `uvicorn`, `sqlalchemy`, `jinja2`, `python-multipart`
- FastAPI 애플리케이션 실행 시 `http://localhost:8000`에서 서버가 동작해야 한다.
- 터미널 로그에서 서버가 정상 기동되었음을 확인할 수 있어야 한다.

## 2. 프로젝트 구조
프로젝트 구조가 최소한 다음 역할을 분리한다.
- `routers/` (요청/응답 처리 — FastAPI Router)
- `services/` (비즈니스 로직)
- `repositories/` (데이터 접근)
- `models/` (SQLAlchemy ORM 모델 클래스)
- 템플릿 파일은 `templates/` 디렉터리 아래에 존재해야 한다.
- `requirements.txt` 또는 `pyproject.toml`에 의존성 목록이 명시되어야 한다.

## 3. 화면 렌더링 (SSR)
- 모든 주요 화면은 Jinja2 템플릿 렌더링 결과(`TemplateResponse`)로 출력되어야 한다.

## 4. 홈 화면
- **URL:** `GET /`
- 홈 화면에는 앱의 목적을 설명하는 문장 1~2줄이 포함되어야 한다.
- 홈 화면에는 최소 2개 이상의 링크(예: 목록, 등록 화면)가 존재해야 한다.

## 5. 도메인 CRUD 기능 (단일 모델 1개)
- 학습자가 스스로 하나의 관리 대상을 정한다.
- "서비스 주제 (데이터 종류)"는 학습자가 직접 정하되, 단일 모델 CRUD가 가능한 수준으로 제한한다. 아래는 참고할 수 있는 주제 및 필드 예시이다.
  - **메모 관리:** 제목, 내용, 작성일시
  - **할 일 관리:** 제목, 완료 여부, 마감일
  - **도서 관리:** 제목, 저자, 출판년도
  - **일기장:** 제목, 내용, 작성일, 기분
  - **단어장:** 단어, 뜻, 예문
- **도메인 선택 가이드:** 모델의 필드는 3~6개 범위를 권장하며, 화면은 `홈 / 목록 / 상세 / 등록(폼) / 수정(폼)` 최소 5개 흐름으로 구성한다.
- 아래 동작이 모두 가능해야 한다.
  - 목록 조회
  - 단건 조회 (상세)
  - 등록
  - 수정
  - 삭제
- 상세 화면에는 해당 데이터의 전체 필드가 표시되어야 한다.
- 수정 화면으로 이동하는 링크(또는 버튼), 삭제를 수행하는 링크(또는 버튼), 목록으로 돌아가는 링크가 존재해야 한다.
- 존재하지 않는 데이터 조회 시 적절한 안내 화면(예: "해당 데이터를 찾을 수 없습니다")이 표시되거나, 목록 화면으로 이동해야 한다.

## 6. 폼 기반 입력/수정 처리
- 등록/수정은 브라우저의 HTML Form을 통해 이루어진다.
- 등록/수정 폼에는 입력 필드(`input`/`textarea` 등)와 저장 버튼이 존재해야 하며, 목록으로 돌아갈 수 있는 취소 링크가 포함되어야 한다.
- 등록/수정 요청은 반드시 `POST` 방식으로 처리된다.
- 폼 데이터는 FastAPI의 `Form()` 파라미터로 수신한다.

## 7. Redirect 적용
- 등록/수정/삭제 이후에는 화면이 바로 렌더링되지 않고, 다른 화면으로 리다이렉트되어야 한다.
- `RedirectResponse`를 사용하며, POST 이후 리다이렉트 시 `status_code=303`을 적용한다.
- 새로고침(F5)으로 인해 같은 등록/수정/삭제가 중복 수행되지 않아야 한다.

## 8. 레이어 분리
- 라우터는 요청/응답과 화면 전환에 집중해야 한다.
- 비즈니스 로직은 서비스에 존재해야 한다.
- DB 접근은 저장소(Repository)에 존재해야 한다.

## 9. SQLite + SQLAlchemy 기반 저장
- 데이터는 서버 메모리(리스트/딕셔너리)만으로 관리하면 안 된다.
- SQLite DB 연결이 되어 있어야 하며, CRUD가 DB를 기준으로 동작해야 한다.
- SQLAlchemy ORM의 `Session`을 사용하여 저장/조회가 가능해야 한다.
- DB 세션 관리는 FastAPI의 `Depends`를 활용한 의존성 주입 방식을 사용한다.
- DB 파일(예: `database.db`)이 프로젝트 디렉터리에 생성되어야 하며, DB 브라우저(DB Browser for SQLite 등) 또는 Python 스크립트를 통해 저장된 데이터를 직접 확인할 수 있어야 한다.

## 10. 제출물 구성
- 전체 프로젝트 소스코드 (GitHub 저장소 링크 또는 압축 파일)
- 실행 방법을 설명하는 `README.md` 파일 (가상환경 생성, 패키지 설치, 서버 실행 순서 포함)
- (선택) 주요 화면 스크린샷

# 보너스 과제 (선택)

## 1. 간단한 검색 기능 추가
- 목록 화면에서 특정 조건(예: 제목 포함)으로 필터링할 수 있는 흐름을 추가해 본다.
- **배움 포인트:** 쿼리 파라미터를 활용한 GET 요청 처리와 SQLAlchemy의 `filter`/`like` 쿼리를 경험한다.

## 2. 간단한 검증 처리
- 필수값이 비어 있으면 저장되지 않도록 하고, 화면에 안내 문구를 보여준다.
- **배움 포인트:** 서버 사이드 검증의 기본 흐름과 Pydantic validator 또는 수동 검증 방식을 경험한다.

## 3. 수정/삭제 UX 개선
- 상세 화면에서 수정/삭제로 자연스럽게 이어지도록 링크/버튼을 설계해 본다.
- **배움 포인트:** 화면 간 네비게이션 흐름 설계와 사용자 경험 개선을 경험한다.

# 결과 예시
아래는 정답이 아니라 참고 예시다. 실제 문구, 디자인은 얼마든지 달라도 된다.

### 홈 화면 예시 (GET /)
```text
나의 메모 앱
메모를 등록하고 목록/상세/수정/삭제할 수 있습니다.
[메모 목록 보기] [새 메모 작성]
```

### 목록 화면 예시
```text
메모 목록
- 오늘 배운 것 정리     [상세보기]
- 내일 할 일             [상세보기]
[새 메모 작성]
```
