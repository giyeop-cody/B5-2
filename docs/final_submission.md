# 최종 제출 및 결과물 안내 (Final Submission)

## 1. 최종 결과물 스펙
다음 기능이 정상 동작하는 FastAPI 웹 애플리케이션 1개를 완성한다.

### 1.1. GET / (홈)
- 브라우저에 "이 앱이 무엇을 하는지"를 설명하는 간단한 홈 화면이 출력된다.
- 홈 화면에는 아래 기능 화면으로 이동할 수 있는 링크가 존재한다.

### 1.2. 단일 도메인(예: Memo/Note/Task 등) CRUD 화면 흐름
- 사용자가 브라우저에서 데이터를 등록하고, 목록을 보고, 상세를 보고, 수정하고, 삭제할 수 있다.
- 화면은 Jinja2 템플릿 기반 SSR로 렌더링된다.

### 1.3. POST → Redirect → GET 흐름 적용
- 등록/수정/삭제 이후, 사용자가 새로고침해도 중복 요청이 발생하지 않도록 Redirect 흐름이 적용되어 있다.

### 1.4. 레이어 분리 구조
- 라우터(요청/응답), 서비스(비즈니스 로직), 저장소(데이터 접근)가 역할에 맞게 분리되어 있다.

### 1.5. SQLite + SQLAlchemy 연결 및 동작
- 인메모리 저장(리스트/딕셔너리)이 아니라 SQLite DB에 저장되는 CRUD가 동작한다.
- 단일 모델 1개(예: Memo) 기준으로 ORM 기반 저장/조회가 가능하다.
