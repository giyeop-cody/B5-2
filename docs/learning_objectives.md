# 미션 소개 및 과제 목표 (Learning Objectives)

## 1. 미션 소개
FastAPI가 빠르게 성장하는 이유는 단순히 빠르기 때문만이 아닙니다. 라우터, 서비스, 저장소와 같이 역할이 왜 나눠지는지 이해하면, 다른 어떤 프레임워크를 만나도 같은 사고로 접근할 수 있습니다. 브라우저 요청이 DB까지 갔다가 화면으로 돌아오는 전체 흐름을 직접 구현합니다.

FastAPI는 Python 기반 웹 애플리케이션을 개발할 때 빠르게 성장하고 있는 현대적 웹 프레임워크입니다. 요청 처리, 화면 렌더링, 데이터 저장 등 웹 서비스의 핵심 구성 요소를 직관적인 구조로 다룰 수 있도록 설계되어 있습니다.

이 미션에서는 FastAPI와 SQLAlchemy를 활용해 하나의 웹 애플리케이션을 처음부터 끝까지 직접 구현합니다. 단순 기능 구현이 아니라, 브라우저 요청이 서버 내부를 거쳐 데이터베이스에 저장되고 다시 화면으로 전달되는 전체 흐름을 경험합니다. 또한, 라우터, 서비스, 저장소로 역할을 분리하고, PRG 패턴과 ORM 기반 CRUD를 적용함으로써 웹 애플리케이션이 유지보수 가능하게 설계되는 이유와 구조적 장점을 이해하게 됩니다.

이 경험은 이후 인증/인가, 모델 간 연관관계 등 고급 주제로 확장되며, 다른 프레임워크(Django, Flask 등)를 학습할 때에도 동일한 구조적 사고(레이어 분리, 요청-응답 흐름 설계)를 적용할 수 있는 기반이 됩니다.

---

## 2. 과제 목표
이 과제를 마친 후, 학습자는 아래를 스스로 설명할 수 있어야 한다.

- 브라우저 요청이 들어왔을 때 FastAPI 서버에서 "라우터 → 서비스 → 저장소 → 템플릿(뷰)"로 흐르는 과정을 말로 설명할 수 있다.
- GET과 POST의 역할 차이를 구분하고, 왜 분리해서 써야 하는지 설명할 수 있다.
- 폼 입력값이 서버로 전달되고, Pydantic 모델 또는 Form 파라미터로 다뤄지는 과정을 설명할 수 있다.
- PRG(Post-Redirect-Get) 패턴을 적용해야 하는 이유를 설명할 수 있다.
- SQLAlchemy ORM으로 단일 모델 CRUD가 동작하는 원리를 "내 코드 구조"를 근거로 설명할 수 있다.

## 3. 보너스 과제를 통한 확장 배움 포인트 (선택)
보너스 과제를 수행할 경우 다음 핵심 포인트들을 추가로 경험하고 설명할 수 있게 된다.

- **검색 기능:** 쿼리 파라미터를 활용한 GET 요청 처리와 SQLAlchemy의 `filter`/`like` 쿼리를 경험한다.
- **검증 처리:** 서버 사이드 검증의 기본 흐름과 Pydantic validator 또는 수동 검증 방식을 경험한다.
- **UX 개선:** 화면 간 네비게이션 흐름 설계와 사용자 경험 개선을 경험한다.
